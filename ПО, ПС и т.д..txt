// Программа - реализация алгоритма.
// ПО - документация для неё.
// ПП - программа, подлежащая для передачи пользователя (о цене не говорится).

// Сопровождаемость программного средства - совокупность свойств программного средства, которое характеризует усилия, необходимые для его модификации.

// Подхарактеристиками сопровождаемости, согласно ГОСТ 28806-90, являются:

// 1) Модифицируемость - совокупность свойств ПС (программного средства), характеризующее усилия, необходимые для внесения в него изменений,
// связанных с устранением дефектов или приведением в соответствие с изменившейся средой функционирования.
// К этой подхаракетристике относится модульность и средство АО (автоматизированного обновления), SRM (средства работы с клиентами).

// 2) Анализируемость - совокупность свойств ПС (программного средства), характеризующее усилия, необходимые для выявления недостатков ПС или причин его отказов,
// либо для установления частей, которые должны быть видоизменены.
// К этой подхарактеристике относятся те же особенности что и к модифицируемости, плюс система ведения логов, обработчик исключительных ситуаций, отправка отчётов сопроводителю о проблеме.
// IEEE 754

// 3) Стабилизированность - совокупность свойств ПС (программного средства), характеризующее степень отсутствия риска того, внесение в ПС изменений даст непредвиденные проявления.
// Соблюдение стандартов кодирования в части соблюдения модульности не использования глобальных переменных и устаревших компонентов языка.
// ПО, которое протестированно, необходимо хранить там, где будет система управления версиями, чтобы была возможность переключаться между версиями.
// Тестируемость - совокупность свойств ПС, характеризующая усилия, необходимые для его проверки после проведения какого-либо видоизменения.
// Количество
